# 플랜별 보장급부 비호환 이슈

마치 수영/육상 선수가 서로 종목이 다르므로 경쟁을 할수 없는 것과 비슷한 상황이 플랜간에 발생하는지,

그리고 서로 종목이 비슷하여 경쟁이 가능하므로 같은 경기 엔트리에 묶을수 있는 플랜들에 대한 분석을 위해 아래 쿼리를 작성하여 데이터를 시각화함

```sql
   DECLARE @cols AS NVARCHAR(MAX);
   DECLARE @query AS NVARCHAR(MAX);

   -- 1. 피벗할 컬럼(플랜 이름) 리스트 동적 생성
   SELECT @cols = STUFF((
       SELECT ',' + QUOTENAME(p.plan_name)
       FROM mmlfcp2.dbo.TB_MMLFCP_PLAN p
       WHERE EXISTS (
           SELECT 1 
           FROM mmlfcp2.dbo.TB_MMLFCP_PLAN_COVERAGE pc 
           WHERE pc.plan_id = p.plan_id 
             AND pc.use_yn = 'Y'
       )
       GROUP BY p.plan_name
       ORDER BY p.plan_name
       FOR XML PATH(''), TYPE
   ).value('.', 'NVARCHAR(MAX)'), 1, 1, '');

   -- 2. 동적 쿼리 문자열 생성
   -- coverage_cd를 포함하여 조회하고 정렬합니다.
   SET @query = '
   SELECT coverage_cd, coverage_name, ' + @cols + '
   FROM
   (
       SELECT 
           c.coverage_cd,   -- 정렬을 위해 코드 컬럼 추가
           c.coverage_name,
           p.plan_name,
           ''Y'' AS use_flag
       FROM mmlfcp2.dbo.TB_MMLFCP_PLAN_COVERAGE pc
       INNER JOIN mmlfcp2.dbo.TB_MMLFCP_PLAN p 
           ON pc.plan_id = p.plan_id
       INNER JOIN mmlfcp2.dbo.TB_MMLFCP_COVERAGE c 
           ON pc.coverage_cd = c.coverage_cd
       WHERE pc.use_yn = ''Y''
   ) src
   PIVOT
   (
       MAX(use_flag)
       FOR plan_name IN (' + @cols + ')
   ) pvt
   ORDER BY coverage_cd;'; -- 정렬 기준 변경

   -- 3. 쿼리 실행
   EXECUTE(@query);
   ### 변경 사항 설명
   *   **`SELECT c.coverage_cd` 추가**: 안쪽 서브쿼리(`src`)에 `coverage_cd`를 추가했습니다. PIVOT은 집계 함수(`MAX`)에 사용되지 않고, `FOR ... IN` 절에도 없는 컬럼들을 자동으로 그룹화(Row Header) 기준으로 사용합니다. 따라서 `coverage_cd`와 `coverage_name`이 함께 행의 기준이 됩니다.
   *   **`ORDER BY coverage_cd`**: 최종 결과의 정렬 순서를 담보 코드 기준으로 변경했습니다.
   *   **`SELECT coverage_cd, ...`**: 결과 맨 앞 열에 코드를 함께 출력하도록 하여, 정렬이 올바르게 되었는지 직관적으로 확인할 수 있게 했습니다.
```
